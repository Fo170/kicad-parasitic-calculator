KiCad dispose d'un système de plugins/scripting bien documenté, bien que le niveau de difficulté dépende de ce que tu veux accomplir :
Deux approches principales
1. Python Scripting (Action Plugins) — Le plus accessible
KiCad expose une API Python (pcbnew pour le routage, eeschema pour le schéma)
Tu peux créer des plugins qui apparaissent directement dans le menu Outils → Plugins externes
Idéal pour : génération automatique de footprints, vérifications DRC custom, export de données, placement automatique de composants, etc.
Exemple minimal Python :


import pcbnew

class MonPlugin(pcbnew.ActionPlugin):
    def defaults(self):
        self.name = "Mon Plugin"
        self.category = "Utilitaires"
        self.description = "Fait quelque chose d'utile"
    
    def Run(self):
        board = pcbnew.GetBoard()
        # Ton code ici
        print(f"PCB: {board.GetFileName()}")
2. Plugins C++ natifs — Pour des performances maximales
Nécessite de compiler KiCad depuis les sources
Réservé aux modifications profondes du comportement interne
Où trouver la doc et des exemples
Documentation officielle : KiCad Python API Reference
Dépôt d'exemples : KiCad / kicad-source-mirror — voir le dossier demos/python_scripts/
Plugins communautaires : KiCad PCM (Plugin and Content Manager) — tu peux aussi publier ton plugin là-bas
Contraintes à connaître

Table
Aspect	Détail
Stabilité API	L'API Python évolue entre les versions majeures (v6 → v7 → v8), donc maintenance nécessaire
Documentation	Parfois lacunaire ; il faut souvent lire le code source C++ ou tester dans la console Python intégrée
Portée	L'API Python ne donne pas accès à tout (certaines fonctions internes restent en C++ uniquement)
En résumé
Si tu maîtrises déjà Python, c'est relativement simple pour des plugins utilitaires. Si tu veux modifier le cœur de KiCad ou faire de l'UI complexe, cela demande plus d'investissement.

Défis techniques spécifiques
1. Extraction du chemin de piste Python :

# Récupérer les segments de piste d'un net
track_list = [t for t in board.GetTracks() if t.GetNetCode() == net_code]

# Trouver les vias
vias = [t for t in track_list if isinstance(t, pcbnew.PCB_VIA)]
2. Calcul de la longueur effective
KiCad te donne les segments (lignes, arcs), tu dois reconstruire le chemin
Attention aux couches : une piste peut changer de couche via un via (impédance différente)
3. Modèle de retour de courant
L'inductance dépend du chemin de retour (généralement le plan de masse le plus proche)
Sans connaître le chemin de retour exact, tu peux faire une approximation avec la distance au plan de masse
4. Capacité parasite
Capacité piste-piste (couplage) vs piste-plan de masse (dominant)
Pour une piste simple au-dessus d'un plan de masse : modèle de microstrip ou stripline selon la couche

class ParasiticCalculator:

        self.rho = 1.68e-8 * (1 + 0.0039*(temp_c - 20))  # Cuivre

        area = width_m * self.thickness

Recommandation d'approche
Commencer simple : calcul de R uniquement (le plus facile, données directes)
Ajouter L : modèle microstrip basique avec plan de masse infini sous la piste
Ajouter C : capacité piste-plan de masse
Affiner : prise en compte des changements de couche, couplage adjacent, etc.
